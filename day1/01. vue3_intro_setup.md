## 🧭 1. Vue.js란?
### Vue.js 아키텍처

Vue.js는 MVVM(Model-View-ViewModel) 패턴을 기반으로 하는 프론트엔드 프레임워크입니다.
### Vue 3가 MVVM을 따르는 방식

| MVVM 구성요소 | Vue 3에서 대응되는 부분 |
| :--- | :--- |
| Model | `data`, `state`, `props`, `reactive`, `ref` |
| View | 템플릿(`<template>`) |
| ViewModel | Vue 인스턴스 또는 Composition API의 `setup()` 함수 |

#### 예시

**Counter.vue**

```vue
<template>
  <div>
    <h1>카운터: {{ count }}</h1>
    <button @click="increment">증가</button>
  </div>
</template>

<script setup>
import { ref } from 'vue'

// Model
const count = ref(0)

// ViewModel: 비즈니스 로직
function increment() {
  count.value++
}
</script>


```

### 구성 분석
- **Model**: `count` 변수 (`ref` 사용)
- **View**: `<template>`에 의해 정의
- **ViewModel**: `setup()` 내부의 로직

## 요약

- Vue 3 컴포넌트 = **ViewModel**
- Vue의 반응형 시스템은 **Model**의 변경을 **View**에 자동 반영
- `setup()` 함수 안에서 상태(state)와 로직을 정의하여 MVVM 구조를 명확하게 구현

#### 1. 핵심 아키텍처 구성요소

- **Virtual DOM**: 실제 DOM과 동기화되는 메모리 내 가상 DOM 트리를 사용하여 성능을 최적화
- **반응형 시스템**: 데이터 변경을 자동으로 감지하고 UI를 업데이트
- **컴포넌트 기반**: 재사용 가능한 독립적인 UI 단위로 구성

#### 2. 주요 아키텍처 특징

| 구성요소 | 설명 |
|---------|------|
| 템플릿 엔진 | HTML 기반의 선언적 렌더링 지원 |
| 반응형 시스템 | Proxy 기반의 향상된 반응성 제공 |
| 컴파일러 | 템플릿을 최적화된 렌더링 함수로 변환 |
| 런타임 | 컴포넌트 라이프사이클과 렌더링 관리 |

#### 3. 데이터 흐름

- Vue.js는 사용자 인터페이스 구축을 위한 **진보적인 JavaScript 프레임워크**입니다.
- 핵심 라이브러리는 **뷰(View) 레이어**에만 집중되어 있으며, 점진적으로 도입이 가능합니다.
- Vue 3는 Composition API, 향상된 성능, 개선된 TypeScript 지원 등 많은 변화가 포함되어 있습니다.

## 🚀 2. Vue 3의 주요 특징

| 기능             | 설명 |
|------------------|------|
| Composition API  | 로직 재사용성과 컴포넌트 구조 개선 |
| 가벼운 번들 사이즈 | Vue 2 대비 약 50% 더 작아짐 |
| 더 빠른 렌더링 성능 | 가상 DOM 개선 및 최적화 |
| 프래그먼트, 텔레포트, Suspense 지원 | 컴포넌트 구조 유연성 강화 |
| TypeScript 공식 지원 | 강력한 타입 추론 및 IDE 자동완성 지원 |

---
</br>

> Composition API란?
- Vue 3에서 도입된 새로운 방식의 API입니다
기존 Options API의 단점을 보완하고 코드 재사용성과 유지보수성을 향상시킵니다
- setup() 함수를 중심으로 컴포넌트의 로직을 구성합니다
- 로직 재사용성: 관련된 로직을 함수로 분리하여 여러 컴포넌트에서 재사용 가능
- 타입 추론: TypeScript와의 더 나은 통합 지원
- 코드 구조화: 관련 로직을 함께 그룹화하여 관리 가능

#### 패키지 관리자 선택

Vue 프로젝트에서 사용할 수 있는 주요 패키지 관리자들은 다음과 같습니다:

| 패키지 관리자 | 특징 |
|--------------|------|
| npm | - Node.js의 기본 패키지 관리자<br>- 가장 널리 사용됨<br>- `package-lock.json`으로 의존성 관리 |
| yarn | - Facebook이 개발한 대안 패키지 관리자<br>- npm보다 빠른 설치 속도<br>- `yarn.lock`으로 더 안정적인 의존성 관리 |
| pnpm | - 디스크 공간 효율적 사용<br>- 심링크 대신 하드링크 사용<br>- 빠른 설치 속도와 적은 디스크 사용량 |
| yarn berry | - Yarn의 최신 버전(v2+)<br>- Plug'n'Play(PnP) 시스템으로 node_modules 제거<br>- 향상된 캐싱과 의존성 관리 |
