# Vue 3 컴포넌트 설계와 재사용성

## 📌 개요

이 자료는 Vue 3 기반 프로젝트에서 컴포넌트를 **효율적으로 설계하고 재사용성**을 극대화하는 방법에 대해 설명합니다.

---

## 1. 컴포넌트 설계 원칙

### 1.1 단일 책임 원칙 (Single Responsibility Principle)

- 하나의 컴포넌트는 하나의 목적만 가져야 합니다.
- 복잡한 UI는 여러 개의 소형 컴포넌트로 나누어 구성합니다.

### 1.2 명확한 Props와 Emits

- `props`는 외부 데이터를 수신합니다.
- `emits`는 이벤트를 명시적으로 선언합니다.

```js
defineProps<{
  title: string
}>()

defineEmits<{
  (e: 'update:title', newTitle: string): void
}>()
```

## 2. 컴포넌트 구조화 전략

### 2.1 Atomic Design 접근법

| 레벨            | 설명                                                          |
| --------------- | ------------------------------------------------------------- |
| Atom(원자))     | 버튼, 입력창 등 최소 단위 컴포넌트                            |
| Molecule(분자)) | Atom들의 조합 (e.g., 검색 폼, 카드 등)                        |
| Organism        | 분자들의 조합으로 복잡한 구조 (e.g., 헤더, 푸터, 사이드바 등) |
| Template        | 레이아웃 구조 정의                                            |
| Page            | 실제 콘텐츠가 삽입된 화면                                     |

```
src/
├── components/
│   ├── atoms/
│   ├── molecules/
│   ├── organisms/
```

### 2.2 기능별 구조화

프로젝트를 기능 또는 도메인별로 구조화하는 방식.

- 파일의 역할(컴포넌트, 페이지, 훅, 유틸 등)별로 디렉토리를 구분합니다.

```
src/
├── components/
│   ├── auth/
│   ├── dashboard/
│   ├── profile/
│   └── shared/
```

## 3. 재사용 가능한 컴포넌트 작성

### 3.1 Slots 활용

슬롯은 Vue 컴포넌트에서 재사용성을 높이는 강력한 기능입니다. 다양한 슬롯 사용 방법을 살펴보겠습니다:

3.1.1 **기본 슬롯 (Default Slots)**:
```html
<!-- BaseCard.vue -->
<template>
  <div class="card">
    <slot>기본 내용</slot>
  </div>
</template>

<!-- 사용 예시 -->
<BaseCard>
  <p>이 내용이 슬롯에 들어갑니다</p>
</BaseCard>
```

3.1.2. **이름이 있는 슬롯 (Named Slots)**:
```html
<!-- BaseLayout.vue -->
<template>
  <div class="container">
    <header>
      <slot name="header">기본 헤더</slot>
    </header>
    
    <main>
      <slot>기본 내용</slot>
    </main>
    
    <footer>
      <slot name="footer">기본 푸터</slot>
    </footer>
  </div>
</template>

<!-- 사용 예시 -->
<BaseLayout>
  <template #header>
    <h1>여기에 헤더 내용</h1>
  </template>
  
  <p>메인 내용입니다</p>
  
  <template #footer>
    <p>여기에 푸터 내용</p>
  </template>
</BaseLayout>
```

3.1.3. **스코프가 있는 슬롯 (Scoped Slots)**:
```html
<!-- TodoList.vue -->
<template>
  <ul>
    <li v-for="item in items" :key="item.id">
      <slot :item="item" :index="index">
        {{ item.text }}
      </slot>
    </li>
  </ul>
</template>

<script setup>
const items = [
  { id: 1, text: '할 일 1' },
  { id: 2, text: '할 일 2' }
]
</script>

<!-- 사용 예시 -->
<TodoList>
  <template #default="{ item, index }">
    <span :class="{ done: item.done }">
      {{ index + 1 }}. {{ item.text }}
    </span>
  </template>
</TodoList>
```

3.1.4. **동적 슬롯 이름 (Dynamic Slot Names)**:
```html
<!-- DynamicSlot.vue -->
<template>
  <div>
    <slot :name="slotName"></slot>
  </div>
</template>

<script setup>
const slotName = ref('header')
</script>

<!-- 사용 예시 -->
<DynamicSlot>
  <template #[slotName]>
    동적 슬롯 내용
  </template>
</DynamicSlot>
```

3.1.5. **슬롯 props**:
```html
<!-- FancyList.vue -->
<template>
  <div class="fancy-list">
    <slot 
      v-for="item in items" 
      :key="item.id"
      :item="item"
      :is-selected="selectedId === item.id"
    ></slot>
  </div>
</template>

<script setup>
const items = ref([
  { id: 1, text: '항목 1' },
  { id: 2, text: '항목 2' }
])
const selectedId = ref(1)
</script>

<!-- 사용 예시 -->
<FancyList>
  <template #default="{ item, isSelected }">
    <div :class="{ selected: isSelected }">
      {{ item.text }}
    </div>
  </template>
</FancyList>
```

3.1.6. **슬롯 폴백 (Fallback) 내용**:
```html
<!-- Button.vue -->
<template>
  <button class="btn">
    <slot>
      <span class="default-icon">→</span>
      기본 버튼
    </slot>
  </button>
</template>

<!-- 사용 예시 -->
<Button>커스텀 내용</Button>
<Button></Button> <!-- 기본 내용이 표시됨 -->
```

#### 슬롯의 주요 장점:
1. **재사용성**: 컴포넌트의 구조는 유지하면서 내용만 변경 가능
2. **유연성**: 부모 컴포넌트에서 자식 컴포넌트의 내용을 커스터마이즈 가능
3. **구성 가능성**: 여러 슬롯을 조합하여 복잡한 레이아웃 구성 가능
4. **데이터 전달**: 스코프가 있는 슬롯을 통해 자식 컴포넌트의 데이터를 부모 컴포넌트에서 사용 가능

> 이러한 슬롯 기능을 활용하면 더 유연하고 재사용 가능한 컴포넌트를 만들 수 있습니다.

### 3.2 Props의 유연성

Props는 컴포넌트의 재사용성과 유연성을 높이는 핵심 요소입니다. 다양한 props 활용 방법을 살펴보겠습니다:

3.2.1. **기본 Props 정의**:
```html
<!-- BaseButton.vue -->
<template>
  <button 
    :class="[
      `btn-${variant}`,
      { 'btn-disabled': disabled }
    ]"
    :disabled="disabled"
    @click="handleClick"
  >
    <slot>{{ label }}</slot>
  </button>
</template>

<script setup>
const props = defineProps({
  variant: {
    type: String,
    default: 'primary',
    validator: (value) => ['primary', 'secondary', 'danger'].includes(value)
  },
  disabled: {
    type: Boolean,
    default: false
  },
  label: {
    type: String,
    default: '버튼'
  }
})

const emit = defineEmits(['click'])

const handleClick = (event) => {
  if (!props.disabled) {
    emit('click', event)
  }
}
</script>

<style scoped>
.btn-primary {
  background: #4CAF50;
  color: white;
}
.btn-secondary {
  background: #9E9E9E;
  color: white;
}
.btn-danger {
  background: #F44336;
  color: white;
}
.btn-disabled {
  opacity: 0.6;
  cursor: not-allowed;
}
</style>
```

3.2.2. **동적 Props**:
```html

      width: width,
```

3.2.3. **Props 유효성 검사**:
```html
<!-- ValidatedInput.vue -->
<template>
  <div class="input-wrapper">
    <input
      :type="type"
      :value="modelValue"
      @input="handleInput"
      :class="{ 'is-invalid': error }"
    />
    <span v-if="error" class="error-message">{{ error }}</span>
  </div>
</template>

<script setup>
import { ref } from 'vue'

const props = defineProps({
  modelValue: {
    type: String,
    required: true
  },
  type: {
    type: String,
    default: 'text',
    validator: (value) => ['text', 'email', 'password'].includes(value)
  },
  required: {
    type: Boolean,
    default: false
  },
  minLength: {
    type: Number,
    default: 0
  },
  maxLength: {
    type: Number,
    default: Infinity
  },
  pattern: {
    type: String,
    default: ''
  }
})

const emit = defineEmits(['update:modelValue', 'validation'])
const error = ref('')

const validate = (value) => {
  if (props.required && !value) {
    error.value = '이 필드는 필수입니다'
    return false
  }
  if (value.length < props.minLength) {
    error.value = `최소 ${props.minLength}자 이상 입력해주세요`
    return false
  }
  if (value.length > props.maxLength) {
    error.value = `최대 ${props.maxLength}자까지 입력 가능합니다`
    return false
  }
  if (props.pattern && !new RegExp(props.pattern).test(value)) {
    error.value = '올바른 형식이 아닙니다'
    return false
  }
  error.value = ''
  return true
}

const handleInput = (event) => {
  const value = event.target.value
  emit('update:modelValue', value)
  const isValid = validate(value)
  emit('validation', isValid)
}
</script>
```

3.2.4. **Props 전달 최적화**:
```html
<!-- OptimizedList.vue -->
<template>
  <div class="list">
    <div 
      v-for="item in items" 
      :key="item.id"
      class="list-item"
      :class="{ 'is-selected': selectedId === item.id }"
      @click="selectItem(item.id)"
    >
      <slot name="item" :item="item">
        {{ item.name }}
      </slot>
    </div>
  </div>
</template>

<script setup>
defineProps({
  items: {
    type: Array,
    required: true,
    validator: (value) => value.every(item => 'id' in item && 'name' in item)
  },
  selectedId: {
    type: Number,
    default: null
  }
})

const emit = defineEmits(['select'])

const selectItem = (id) => {
  emit('select', id)
}
</script>
```

#### Props 활용의 주요 장점:
- **타입 안정성**: Props 정의를 통한 런타임 타입 체크
   - 컴포넌트가 받을 수 있는 props의 타입을 명시적으로 정의
   - 잘못된 타입의 데이터가 전달되는 것을 방지
   - 예시:
     ```js
     defineProps({
       count: {
         type: Number,  // 숫자 타입만 허용
         required: true // 필수 값으로 지정
       },
       title: {
         type: String,  // 문자열 타입만 허용
         default: ''    // 기본값 설정
       },
       items: {
         type: Array,   // 배열 타입만 허용
         validator: (value) => value.length > 0  // 유효성 검사 추가
       }
     })
     ```
   - 이러한 타입 체크를 통해:
     - 개발 시점에 잘못된 props 사용을 방지
     - 런타임 에러 발생 가능성 감소
     - 컴포넌트 사용 방법을 명확하게 문서화
     - IDE의 자동 완성 기능 지원

- **재사용성**: 다양한 상황에 맞게 컴포넌트 커스터마이징 가능

### 3.3 Composition API 기반 로직 재사용

Composition API를 사용하면 컴포넌트의 로직을 재사용 가능한 함수로 분리할 수 있습니다. 이러한 함수를 'composables'라고 합니다.

#### 3.3.1 기본적인 Composable 예제

1. **토글 기능**:
```js
// composables/useToggle.js
import { ref } from 'vue'

export function useToggle(initial = false) {
  const state = ref(initial)
  const toggle = () => (state.value = !state.value)
  const setTrue = () => (state.value = true)
  const setFalse = () => (state.value = false)
  
  return { 
    state,
    toggle,
    setTrue,
    setFalse
  }
}

// 사용 예시
const { state, toggle } = useToggle()
```

2. **마우스 위치 추적**:
```js
// composables/useMousePosition.js
import { ref, onMounted, onUnmounted } from 'vue'

export function useMousePosition() {
  const x = ref(0)
  const y = ref(0)

  const updatePosition = (event) => {
    x.value = event.pageX
    y.value = event.pageY
  }

  onMounted(() => {
    window.addEventListener('mousemove', updatePosition)
  })

  onUnmounted(() => {
    window.removeEventListener('mousemove', updatePosition)
  })

  return { x, y }
}

// 사용 예시
const { x, y } = useMousePosition()
```

#### 3.3.2 비동기 데이터 처리

```js
// composables/useAsync.js
import { ref } from 'vue'

export function useAsync(asyncFunction) {
  const data = ref(null)
  const error = ref(null)
  const loading = ref(false)

  const execute = async (...args) => {
    loading.value = true
    error.value = null
    
    try {
      data.value = await asyncFunction(...args)
    } catch (e) {
      error.value = e
    } finally {
      loading.value = false
    }
  }

  return {
    data,
    error,
    loading,
    execute
  }
}

// 사용 예시
const fetchUser = async (id) => {
  const response = await fetch(`/api/users/${id}`)
  return response.json()
}

const { data, error, loading, execute } = useAsync(fetchUser)
```

#### 3.3.3 폼 처리

```js
// composables/useForm.js
import { ref, computed } from 'vue'

export function useForm(initialValues = {}, validationRules = {}) {
  const values = ref({ ...initialValues })
  const errors = ref({})
  const touched = ref({})

  const setFieldValue = (field, value) => {
    values.value[field] = value
    validateField(field)
  }

  const setFieldTouched = (field) => {
    touched.value[field] = true
    validateField(field)
  }

  const validateField = (field) => {
    const rule = validationRules[field]
    if (rule) {
      errors.value[field] = rule(values.value[field])
    }
  }

  const validateForm = () => {
    Object.keys(validationRules).forEach(validateField)
    return !Object.values(errors.value).some(error => error)
  }

  const resetForm = () => {
    values.value = { ...initialValues }
    errors.value = {}
    touched.value = {}
  }

  const isFieldValid = (field) => {
    return !errors.value[field]
  }

  const isFieldTouched = (field) => {
    return touched.value[field]
  }

  return {
    values,
    errors,
    touched,
    setFieldValue,
    setFieldTouched,
    validateForm,
    resetForm,
    isFieldValid,
    isFieldTouched
  }
}

// 사용 예시
const { 
  values, 
  errors, 
  setFieldValue, 
  validateForm 
} = useForm(
  { username: '', password: '' },
  {
    username: (value) => !value ? '사용자명은 필수입니다' : '',
    password: (value) => value.length < 6 ? '비밀번호는 6자 이상이어야 합니다' : ''
  }
)
```

#### 3.3.4 Composable 사용의 장점

1. **로직 재사용**
   - 여러 컴포넌트에서 동일한 로직을 쉽게 재사용
   - 코드 중복 방지
   - 유지보수 용이성 향상

2. **관심사 분리**
   - 비즈니스 로직과 UI 로직을 분리
   - 테스트 용이성 향상
   - 코드 가독성 개선

3. **타입 안정성**
   - TypeScript와 함께 사용 시 타입 추론 지원
   - IDE의 자동 완성 기능 활용 가능

4. **테스트 용이성**
   - 순수 함수로 분리되어 단위 테스트 작성 용이
   - 의존성 주입을 통한 테스트 환경 구성 가능

이러한 Composable 패턴을 활용하면 Vue 애플리케이션의 코드 품질과 유지보수성을 크게 향상시킬 수 있습니다.
